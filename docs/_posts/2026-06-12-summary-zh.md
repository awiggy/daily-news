## [AMD RCE 漏洞未修复，仅用 CRC-32 检查](https://mrbruh.com/amd2/) ⭐️ 8.0/10

AMD 软件中的一个远程代码执行（RCE）漏洞仍未修复，其补丁仅添加了 CRC-32 完整性检查，而非正确的加密签名验证，一旦 Web 服务器被攻破，利用该漏洞将变得轻而易举。 这凸显了 AMD 安全实践不足，可能使数百万用户面临远程攻击风险。未能实施正确的签名验证损害了用户对 AMD 软件安全的信任，并可能对企业及消费级系统造成严重后果。 该漏洞允许通过参数注入实现经过身份验证的 RCE，而 AMD 的修复仅使用 CRC-32 检查，该检查不具备加密安全性，可轻易绕过。研究人员指出，虽然 HTTPS 可防止中间人攻击，但一旦 Web 服务器被攻破，利用该漏洞仍然轻而易举。

hackernews · MrBruh · 6月11日 16:03 · [社区讨论](https://news.ycombinator.com/item?id=48492215)

**背景**: CRC-32 是一种用于检测意外数据损坏的错误检测码，但不具备加密安全性，攻击者可轻易伪造。正确的签名验证使用加密哈希（如 SHA-256）和数字签名来确保完整性和真实性。AMD 使用 CRC-32 而非加密签名意味着，攻破 Web 服务器的攻击者可以在保持相同 CRC-32 值的同时，将可执行文件替换为恶意文件。
